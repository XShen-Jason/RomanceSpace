🔌 主平台与 Worker 数据连通 API 指南

这份文档面向 主平台（如 Next.js）开发人员，阐述了如何将您的系统（用户认领域名、挑选模版、配置内容）与 RomanceSpace 的底层边缘引擎（Worker + KV）进行无缝数据连通。

1. 交互全景图

RomanceSpace 的架构是前后端分离的。Next.js 负责所有的复杂业务（登录注册、收款、表单填写），而 Worker 仅作为高速的呈现通道。核心的交互桥梁就是 Cloudflare KV。

  1. 用户在 Next.js 选择一个前缀并声明：“我要 xiaoming.885201314.xyz 这个域名！”
  2. 用户继续操作：“我要把这个域名装上 love_letter (表白信) 这个高级模版，并且写上我想说的话。”
  3. Next.js 将用户的话打包成一段 JSON，偷偷塞入 Cloudflare KV 数据库，贴上标签 xiaoming。
  4. 搞定。此时任何人访问 xiaoming.885201314.xyz 都会瞬间看到刚刚的定制效果。

2. 第一步：初始配置与模版挂载（Create）

当用户在您的 Next.js 系统中完成付款或认领了某个子域名，并且进入后台选定了某个模版后，您需要调用 Cloudflare 的 REST API 将对应的文件塞进 KV 中：

Endpoint (HTTP PUT)

https://api.cloudflare.com/client/v4/accounts/{您的_CF_Account_ID}/storage/kv/namespaces/{您的_KV_Namespace_ID}/values/<用户抢到的域名前缀>

请求头 (Headers)

Authorization: Bearer <你的_Cloudflare_API_Token>
Content-Type: application/json

请求体 (Body Payload)

假设用户抢到的名字叫 xiaoming，挑选了 love_letter 模版：

{
  "type": "love_letter",   // 极其关键：这决定了边缘引擎调用哪个文件夹里的代码进行渲染
  "data": {                // 这个 data 里的所有内容都是你随便定的，它会原封不动地传进目标模版的函数里
      "title": "小红，我喜欢你",
      "sender": "小明",
      "themeColor": "#ff6b81"
  }
}

3. 第二步：用户随时访问所属域名 (Read)

由于我们已经在 Cloudflare 层面配置好了 *.885201314.xyz 的强制通配符 DNS 托管，只要 Next.js 在第一步往 KV 里写入了数据，该用户的域名就立刻宣告生效，无需任何额外的 DNS 解析或服务器重启。

Next.js 只需要在前端给用户抛出一个按钮：“🔗 点此访问你的专属页面：https://xiaoming.885201314.xyz” 即可。剩下的脏活全交给 RomanceSpace Worker 边缘系统高速处理。

4. 第三步：后续资料修改与热更新 (Update)

人的想法总是多变的。当过了几个礼拜，小明登录你的 Next.js 主站后台，说他想把情书里的话改成别的内容：

  • 这不需要调用任何更新（Patch）接口，KV 是简单的键值对。
  • Next.js 服务端只需要把旧的 JSON 拿出来，替换掉他修改的新文案（比如把 sender 改为老王）。
  • 然后再次对最初的那个 Endpoint 发起一模一样的 PUT 覆盖请求。

💡 给独立开发者的建议：封装备库

在 Next.js 的 Server Actions 中，你可以直接封装一个通用的 syncToWorker(username, projectType, payload) 异步函数。凡是在主系统里发生了针对页面内容的增删改，在最后一行调用一下该函数进行云端 PUT 同步即可。这就是业界标准的 Serverless 读写分离架构！
