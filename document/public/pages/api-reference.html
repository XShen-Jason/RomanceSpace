🔌 KV 配置同步 API

本文档说明了主站如何与前端展示页面库（KV）连接并传递数据。

1. 业务交互流程

  1. 注册与选定： 用户注册后如果选择了模板，首次可以领取一个专属的子域名（如 xiaoming.885201314.xyz）。主站向 KV 发送最基础的模版标识（type），页面将使用模板默认值渲染。
  2. 定制内容： 用户升级或使用免费次数想要修改模版内容时（如修改文字、图片），在主站进行表单填写。
  3. 数据更新： 用户提交后，主站发起接口请求，将用户填写的全新定制内容（包含完整的 type 和 data）保存到 KV 的对应域名记录下。网页将在下次访问时即刻展示更新后的内容。

[PUT] 写入/更新配置记录

用于创建新的域名关联，或更新已有子域名的对应展示内容。

请求 URL

PUT https://api.cloudflare.com/client/v4/accounts/{ACCOUNT_ID}/storage/kv/namespaces/{NAMESPACE_ID}/values/{子域名前缀}

请求 Headers

  • Authorization: Bearer [CF_API_TOKEN]
  • Content-Type: application/json

参数 (JSON)

字段 类型 必填 描述
type String 是 模板名代号。如 love_letter 或 anniversary
data Object 否 用户定制的参数数据。不传或传空时，模板将渲染自身的默认内容

示例

{
  "type": "love_letter",
  "data": {
    "title": "To Xiaohong",
    "sender": "Xiaoming",
    "receiver": "Xiaohong",
    "paragraphs": ["You are the apple of my eye.", "I love you!"]
  }
}

响应 HTTP 200 OK

{
  "result": null,
  "success": true,
  "errors": [],
  "messages": []
}
