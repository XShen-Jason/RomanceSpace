🔌 项目的开通与修改接口

本文档说明了主站（中心服务器）如何与前端边缘访问层（基于 Cloudflare）交互以开通和更新用户页面。

🌐 统一 API 请求规范

请求 URL:

PUT https://api.cloudflare.com/client/v4/accounts/{ACCOUNT_ID}/storage/kv/namespaces/{NAMESPACE_ID}/values/{分配给用户的子域名}

请求 Headers:

  • Authorization: Bearer [CF_API_TOKEN]
  • Content-Type: application/json

* 下方的业务场景，均通过调用此接口配合不同的 JSON 参数实现。

⚠️ 核心业务规则 (必读)

  • 先选模板，后绑域名： 用户必须在主平台选定任意一个模板后，才允许其设置专属的三级域名。
  • 主站查重校验： 在请求本接口创建记录前，主平台后端必须先进行域名的查重校验，确保该域名未被占用，随后连同模板信息一并发送给此 API 接口。

场景一：首次开通 (选中模板并启用域名)

业务流程： 用户确定了想要使用的模板（如表白信 love_letter），随后在主站完成了域名的输入与可用性校验。主站调用接口，同时将域名标识和模板标识一并初始化。因无定制内容，平台将渲染模板的默认文本用于预览。

发送参数 (JSON)：

{
  "type": "love_letter"
}

注：此时不传 data，模板使用默认值渲染。

场景二：后续修改参数或更换模板

业务流程： 用户决定深度修改该模板里的各项定制内容，或者干脆直接切换成另一款模板。接口将推送对应的 type 及全量的 data 参数，边缘节点收到后即时重写覆盖旧记录并展现新网页。

发送参数 (JSON)：

{
  "type": "love_letter",
  "data": {
    "title": "To 最爱的你",
    "sender": "小明",
    "receiver": "小红",
    "paragraphs": ["一直在一起吧！这是我写给你的表白信。"]
  }
}
