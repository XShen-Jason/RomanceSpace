🔌 项目的开通与修改接口

本文档说明了主站（中心服务器）如何与前端边缘访问层（基于 Cloudflare）交互以开通和更新用户页面。

🌐 统一 API 请求规范

请求 URL:

PUT https://api.cloudflare.com/client/v4/accounts/{ACCOUNT_ID}/storage/kv/namespaces/{NAMESPACE_ID}/values/{分配给用户的子域名}

请求 Headers:

  • Authorization: Bearer [CF_API_TOKEN]
  • Content-Type: application/json

* 下方的所有业务场景，均通过调用此接口配合不同的 JSON 参数实现。

场景一：启用并初始化子域名

业务流程： 用户注册主站后，领取专属三级域名（如 xiaoming），此环节系统仅对其占位并进行初始化，但不立刻分配任何模板。用户访问将看到平台欢迎或未激活提示。

发送参数 (JSON)：

{
  "type": "uninitialized",
  "data": {}
}

场景二：首次选中模板使用

业务流程： 用户在其域名下选择了某个模板（如表白信 love_letter），但尚未填写任何个性化定制信息。此时调用接口，平台会利用该模板内置的默认文本与布局进行渲染。

发送参数 (JSON)：

{
  "type": "love_letter"
}

注：因为没有包含 data，模板代码将自动使用初始化默认值供用户预览。

场景三：后续修改模板内容 (深度定制)

业务流程： 用户使用了升级权限或免费次数，进入主站后台深度修改了该模板里的文字、名字或图片。接口推送全量的 data 参数后，边缘节点即时生效并展现新网页。

发送参数 (JSON)：

{
  "type": "love_letter",
  "data": {
    "title": "To 亲爱的红红",
    "sender": "小明",
    "receiver": "小红",
    "paragraphs": ["一直在一起吧！这是我写给你的表白信。"]
  }
}
