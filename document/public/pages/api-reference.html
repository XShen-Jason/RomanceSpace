🔌 API 接口对接规范

本文档专为 Next.js 主平台开发者 编写。讲解了主平台如何通过 Cloudflare KV 与 RomanceSpace Worker (边缘渲染引擎) 进行数据联通。

1. 业务全链路流程图解

  1. 挑选与领取 (Next.js)： 用户在主站注册，并选择心仪的模板（如：情书 love_letter），确立自己的专属三级域名前缀（如 xiaoming）。
  2. 初始化填表 (Next.js)： 用户填写对应模板需要的文案、名字、日期等信息。
  3. 数据推库 (Next.js -> CF KV)： Next.js 后端发起 HTTP 请求，将包含 type 和 data 的 JSON 对象存入 Cloudflare KV，键名为 xiaoming。
  4. 终端访问 (Browser -> Worker)： 用户访问 xiaoming.885201314.xyz，Worker 拦截后从 KV 取出数据并渲染出完整网页。
  5. 后续修改 (Next.js -> CF KV)： 用户在主站修改了文案，Next.js 再次调用接口全量覆盖 KV 中的该条记录即可实时生效。
