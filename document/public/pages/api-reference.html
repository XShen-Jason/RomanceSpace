🔌 KV 配置中心同步 API

本文档专为 主平台开发者 编写。讲解了主平台如何通过 Cloudflare KV 与 RomanceSpace Worker (边缘渲染引擎) 进行数据联通。

1. 业务交互流程与预设值逻辑

很多模板在开发时已经内置了**默认值**。用户领取专属域名并分配某款模版时，不需要立刻填写所有信息，页面也能依据默认值正常渲染。

推荐的业务交互流程：

  1. 零配置开通： 用户在主站（Next.js）挑选并领取一个模版（通过免费次数或者购买），获得类似 xiaoming.885201314.xyz 的专属域名。主站此时只需往 KV 发送最基础的 JSON（仅包含 type 或少量必须字段），模版即可呈现默认样貌。
  2. 升级/消耗次数定制： 当用户希望个性化模版（例如上传照片、修改标题、背景音乐等），触发平台上的“去定制/升级”按钮（可能消耗免费配置次数或进行付费）。
  3. 重新入库 (Push to KV)： 用户提交定制表单后，Next.js 主站发起 HTTP 请求，将完整的配置数据覆盖至 KV 中的同名键 `xiaoming` 下，网页即刻生效新内容。

[PUT] 写入/更新用户项目配置

创建或全量覆盖边缘引擎 KV 中的配置记录。

请求 URL

PUT https://api.cloudflare.com/client/v4/accounts/{ACCOUNT_ID}/storage/kv/namespaces/{NAMESPACE_ID}/values/{项目标识(即子域名前缀)}

请求 Headers

  • Authorization: Bearer [CF_API_TOKEN]
  • Content-Type: application/json

Body Params (JSON)

字段 类型 必填 描述
type String 是 关联的模板名。例如 love_letter 或 anniversary
data Object 否 传递给模版的个性化定制数据。不传则代表使用模板自带默认值

Body Payload 示例

{
  "type": "love_letter",
  "data": {
    "title": "To Xiaohong",
    "sender": "Xiaoming",
    "receiver": "Xiaohong",
    "paragraphs": ["You are the apple of my eye.", "I love you forever!"]
  }
}

响应 HTTP 200 OK

{
  "result": null,
  "success": true,
  "errors": [],
  "messages": []
}
