🛠️ RomanceSpace 主引挚 (Worker) 开发与集成指南

这份文档面向后端开发人员与前端页面开发者，阐述了 RomanceSpace 边缘引擎的核心架构，以及外部系统如何进行数据交互。

1. 架构总览

RomanceSpace 的核心用户项目分发基于 Cloudflare Worker 和 Cloudflare KV（键值对存储）。

  • 请求链路示例：
    1. 用户在浏览器输入定制的三级域名：https://xiaoming.885201314.xyz
    2. 由于 Cloudflare 启用了泛解析 *.885201314.xyz，请求被边缘 Worker 拦截。
    3. Worker 从域名中提取出最前方的子域名作为特征标识 xiaoming。
    4. 通过该标识向 KV 数据库检索匹配的配置参数。
    5. 根据取出的 type 动态选择项目代码包，传入参数渲染最终网页并返回。

2. 主平台如何推送数据到 KV

当用户在主站创建并定制了专属页面后，主站服务端需要将最终配置推送到 Cloudflare KV 中供边缘系统读取。

KV 数据结构说明

Key (键名)

决定用户的访问网址。如果键名为 xiaoming，则绑定访问入口为 xiaoming.885201314.xyz。

Value (键值)

必须包含 type 和 data 两个核心字段的 JSON 字符串文本。
用于承载模板标识与用户定制的各个参数。

// KV 中 Value 的结构示例
{
    "type": "anniversary", // 调用的模板代号
    "data": { // 传递给指定模版的定制参数
        "title": "三周年快乐",
        "startDate": "2023-01-01",
        "message": "一直在一起。"
    }
}

主平台交互简要流程

  1. 在主站服务端接收用户的定制表单请求。
  2. 组合包含 type 和 data 的 JSON。
  3. 使用官方 API，向绑定的 KV Namespace 写入记录，键名为用户的子域名前缀。
  4. 重定向或通知用户访问生成的子域名网页。

3. 目录与路由架构

根路由 src/index.js 作为系统总线负责数据搬运与转发。目录结构如下：

/romancespace
├── src
│   ├── index.js          (主路由器：拦截请求 -> 提取子域名 -> 查KV -> 转发数据)
│   └── projects          (独立模版生态系统箱)
│       ├── love_letter   (情书模版目录)
│       └── anniversary   (纪念日模版目录)

主路由数据流向解析：

  1. 如果请求免配置预览链接（如 /preview/xxx），将跳过数据库查询，直接将默认对象交给对应模块。
  2. 如果是有效的三级域名访问，系统拉取 KV 数据，解析出 type 与 data。
  3. 将 data 配置作为参数传入对应模板的 render 函数中，接收返回的纯 HTML 发送给浏览器。

4. 模版接入：如何将第三方网页变为标准模版

外部源码作为新模板接入系统时，只需遵循以下结构放置并对外暴露函数即可：

文件放置与函数包装

  1. 在 romancespace/src/projects/ 下新建文件夹（如 new_project）。
  2. 在该文件夹中创建 index.js。
  3. 所有的 HTML、CSS 和 JS 脚本须作为字符串通过以下标准模板返回：

👇 小项目包装标准参考：

export function render(config) {
    // 1. 接收自定义变量，并设置默认值
    const { 
        title = "默认标题", 
        name = "默认名字" 
    } = config;
    
    // 2. 返回完整的 HTML 字符串。可使用 \${变量名} 注入数据。
    return `
<!DOCTYPE html>
<html>
<head>
    <title>\${title}</title>
    <style> body { background: pink; } </style>
</head>
<body>
    <h1>Hello, \${name}!</h1>
    <script> console.log("running"); </script>
</body>
</html>
    `;
}
