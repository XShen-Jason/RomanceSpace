🛠️ RomanceSpace 主引挚 (Worker) 开发与集成指南

这份文档面向后端开发人员与前端页面开发者，阐述了 RomanceSpace 边缘 Serverless 引擎的核心架构，以及外部系统如何与该引擎进行数据交互。

1. 架构总览

RomanceSpace 的核心用户项目分发基于 Cloudflare Worker 和 Cloudflare KV（键值对存储）。

  • 请求链路举例：
    1. 用户在浏览器输入定制的三级域名：https://xiaoming.885201314.xyz
    2. 由于 Cloudflare 配置了泛解析 *.885201314.xyz，请求被边缘 Worker 拦截接管。
    3. Worker 从域名中提取出最前方的子域名，即特征信息 xiaoming。
    4. 拿着 xiaoming 去高速 KV 数据库检索匹配的配置。
    5. 根据取出的 type 动态选择项目代码包，传入参数渲染最终网页发给用户浏览。
  • 核心优势： 全球边缘节点万秒级快速渲染，项目前后端彻解耦。

2. 主平台 (如 Next.js) 如何派发数据到 KV

当用户在您的 Next.js 主网站上买单并定制了一个专属的纪念日页面后，您的 Next.js 服务端需要将最终配置推送到 Cloudflare KV 中供边缘系统消费。

深入理解 KV 数据库的 K 与 V

什么是 K (Key / 键名)？

K 决定了用户的访问网址。如果设置 K 为 xiaoming，则它死死绑定访问入口 xiaoming.885201314.xyz。

什么是 V (Value / 键值)？

V 是一串必须包含 type 和 data 两个核心字段的 JSON 字符串文本。
在其中，可以任意塞入用户自己填写的图片链接、文字祝福、日期、音乐等参数内容。

// 理想放入 KV 中 V 的结构样貌
{
    "type": "anniversary", // 告诉 Worker 应该去调用 /projects/anniversary 文件夹里的代码
    "data": { // 这些字段全凭你高兴设计，用来传给指定的模版去显示
        "title": "三周年快乐",
        "startDate": "2023-01-01",
        "message": "一直一直喜欢你。"
    }
}

主平台 Next.js 交互简要流程

  1. 在 Next.js 服务端 (Server Actions 或 API Route) 接收用户的定制表单请求。
  2. 拼接成上述这种包裹 type 和 data 的 JSON。
  3. 使用 Cloudflare 提供的官方 SDK/API，往该项目绑定的 KV Namespace 中写入一条记录，记录的键名为分配给他的子域名前缀。
  4. 此时即可重定向或通知用户：专属页面部署成功，请直接访问你的子域名。

3. Worker 目录与路由架构体系

RomanceSpace 的根路由 src/index.js 完全扮演系统总线（Router）角色，它本身没有任何界面逻辑，仅包含数据的搬运与转发。目录结构非常清晰：

/romancespace
├── src
│   ├── index.js          (主路由器：拦截请求 -> 提取子域名 -> 查KV -> 将V转交组件)
│   └── projects          (独立模版生态系统箱)
│       ├── love_letter   (情书模版目录)
│       └── anniversary   (纪念日模版目录)

主路由数据流向解析：

  1. 如果请求的是免配置的展示链接（如 /preview/xxx），则跳过数据库查询。直接把默认数据对象交给目标文件夹模块。
  2. 如果是有效的三级域名访问，例如提取到了 xiaoming，那么系统会把 xiaoming 对应的 KV 数据拉取出来。把 JSON 解码后，拿上它去唤醒对应目录。
  3. 例如，解出 type: anniversary 取出里面的 data: {...}，主路由会将这段配置作为函数参数投喂给 src/projects/anniversary/index.js 的主函数。最终接收一段渲染好的纯 HTML 送给浏览器。

4. 模版接入：如何将第三方网页变为 RomanceSpace 模版

如果您从网上下载了好看的网页源码（或者您自己编写了一个 HTML/JS 小项目），并希望把它作为 RomanceSpace 的一个新模版供用户选择，您只需要负责第一步，剩下的都可以交给 AI 去配置：

您的工作：放置文件并包装函数

  1. 在 romancespace/src/projects/ 下新建一个文件夹，比如 new_project。
  2. 在该文件夹中创建一个 index.js。
  3. 将您所有的 HTML、CSS 和脚本代码，作为字符串通过下面这个标准模版包裹起来：

👇 小项目包装标准模版 (直接复制修改)：

export function render(config) {
    // 1. 这里是你可以接收的变量，设置好默认值
    const { 
        title = "默认标题", 
        name = "默认名字" 
    } = config;
    
    // 2. 务必返回一段完整的 HTML 字符串，所有的内联 CSS 和 JS 都往里面塞
    // 可以使用 \${变量名} 的语法把变量插进网页里
    return `
<!DOCTYPE html>
<html>
<head>
    <title>\${title}</title>
    <style> body { background: pink; } </style>
</head>
<body>
    <h1>Hello, \${name}!</h1>
    <script> console.log("running"); </script>
</body>
</html>
    `;
}

AI 的工作：向引擎注册

当您放好文件后，您只需要告诉 AI：“我刚刚以新模版标准加入了一个叫 new_project 的模板文件夹，请你帮我更新 src/index.js 主文件并且把它注册到路由中。”

AI 会自动帮您 import 并在 PROJECT_REGISTRY 中建立索引机制，这样该模块就彻底被系统接管可用了。
