🛠️ RomanceSpace 主引挚 (Worker) 开发与集成指南

这份文档面向后端开发人员与前端页面开发者，阐述了 RomanceSpace 边缘 Serverless 引擎的核心架构，以及外部系统如何与该引擎进行数据交互。

1. 架构总览

RomanceSpace 的核心用户项目分发基于 Cloudflare Worker 和 Cloudflare KV（键值对存储）。

  • 请求链路： 用户访问 [项目ID].romancespace.xxx -> 触发 Cloudflare Worker -> 取出子域名 [项目ID] -> 查询 KV 数据库找到配置 -> 根据 type 动态选择项目模版 -> 返回渲染后的 HTML。
  • 优势： 全球边缘节点渲染，极速响应，前后端解耦。

2. 后端开发：远端数据库的 KV 同步 API

当用户在您的主站（如 MySQL/PostgreSQL）购买并定制了一个表白页面或纪念日页面时，您的服务端必须调用 Cloudflare API 将这份数据下发到 KV 存储中。这样边缘节点才能读取到最新配置。

深入理解 KV 数据结构

什么是 KV 的 Key (键名)？

在 Cloudflare KV 中，Key（键名）直接对应于用户将要访问的**专属子域名的前缀**。例如，如果我们设置一个 Key 为 xiaoming，那么系统就会将其绑定到三级域名访问地址：xiaoming.885201314.xyz。

Cloudflare KV Write API

Endpoint (PUT):

https://api.cloudflare.com/client/v4/accounts/{ACCOUNT_ID}/storage/kv/namespaces/{NAMESPACE_ID}/values/{项目标识(即子域名前缀，如xiaoming)}

Headers:

Authorization: Bearer [您的 CF API Token]
Content-Type: application/json

Body (Payload 规范 - 也就是 KV 的 Value):

KV 的 Value (值) 必须是一个包含 type 和 data 属性的 JSON 字符串。type 用于决定加载哪个页面的代码模板，而 data 是真正传递给页面渲染器的数据对象。

{
    "type": "love_letter", // 对应 src/projects/ 下的文件夹名称
    "data": { // 任意传递给该模版的 JSON 数据参数
        "title": "To Xiaohong",
        "sender": "Xiaoming",
        "receiver": "Xiaohong",
        "paragraphs": ["You are the apple of my eye.", "I love you forever!"]
    }
}

3. Worker 主路由架构解析 (src/index.js)

我们的 src/index.js 已经被剥离为一个纯粹的**“交通枢纽” (Router)**，它不包含任何具体页面的 UI 代码。它的执行流程如下：

// 1. 域名分流 (Domain Splitting)
// 如果访问的是 /preview/xxx，跳过数据库，直接调取本地模版进行强制预览。
// 如果访问的是 document.885201314.xyz，原样转发给 Pages（文档中心）。

// 2. 提取特征 (Subdomain Extraction)
// 从 xiaoming.885201314.xyz 中提取出 'xiaoming' 作为查库依据。

// 3. 查阅数据 (KV Fetching)
let projectConfigJSON = await env.ROMANCESPACE_KV.get('xiaoming');

// 4. 模块挂载 (Module Mounting)
const config = JSON.parse(projectConfigJSON);
const templateModule = PROJECT_REGISTRY[config.type]; // 取出对应的代码包
const htmlStream = templateModule.render(config.data); // 将变量塞入包中生成网页
return new Response(htmlStream);

4. 模版接入：如何将第三方网页变为 RomanceSpace 模版

如果您从网上下载了好看的网页源码（或者您自己编写了一个 HTML/JS 小项目），并希望把它作为 RomanceSpace 的一个新模版供用户选择，您只需要负责第一步，剩下的都可以交给 AI 去配置：

您的工作：放置文件并包装函数

  1. 在 romancespace/src/projects/ 下新建一个文件夹，比如 new_project。
  2. 在该文件夹中创建一个 index.js。
  3. 将您所有的 HTML、CSS 和脚本代码，作为字符串通过下面这个标准模版包裹起来：

👇 小项目包装标准模版 (直接复制修改)：

export function render(config) {
    // 1. 这里是你可以接收的变量，设置好默认值
    const { 
        title = "默认标题", 
        name = "默认名字" 
    } = config;
    
    // 2. 务必返回一段完整的 HTML 字符串，所有的内联 CSS 和 JS 都往里面塞
    // 可以使用 \${变量名} 的语法把变量插进网页里
    return `
<!DOCTYPE html>
<html>
<head>
    <title>\${title}</title>
    <style> body { background: pink; } </style>
</head>
<body>
    <h1>Hello, \${name}!</h1>
    <script> console.log("running"); </script>
</body>
</html>
    `;
}

AI 的工作：向引擎注册

当您放好文件后，您只需要告诉 AI：“我刚刚以新模版标准加入了一个叫 new_project 的模板文件夹，请你帮我更新 src/index.js 主文件并且把它注册到路由中。”

AI 会自动帮您 import 并在 PROJECT_REGISTRY 中建立索引机制，这样该模块就彻底被系统接管可用了。
