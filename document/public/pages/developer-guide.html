🛠️ RomanceSpace 主引挚 (Worker) 开发与集成指南

这份文档面向后端开发人员与前端页面开发者，阐述了 RomanceSpace 边缘 Serverless 引擎的核心架构，以及外部系统如何与该引擎进行数据交互。

1. 架构总览

RomanceSpace 的核心用户项目分发基于 Cloudflare Worker 和 Cloudflare KV（键值对存储）。

  • 请求链路： 用户访问 [项目ID].romancespace.xxx -> 触发 Cloudflare Worker -> 取出子域名 [项目ID] -> 查询 KV 数据库找到配置 -> 根据 type 动态选择项目模版 -> 返回渲染后的 HTML。
  • 优势： 全球边缘节点渲染，极速响应，前后端解耦。

2. 后端开发：远端数据库的 KV 同步 API

当用户在您的主站（如 MySQL/PostgreSQL）购买并定制了一个表白页面或纪念日页面时，您的服务端必须调用 Cloudflare API 将这份数据下发到 KV 存储中。这样边缘节点才能读取到最新配置。

Cloudflare KV Write API

Endpoint (PUT):

https://api.cloudflare.com/client/v4/accounts/{ACCOUNT_ID}/storage/kv/namespaces/{NAMESPACE_ID}/values/{项目标识(如xiaoming)}

Headers:

Authorization: Bearer [您的 CF API Token]
Content-Type: application/json

Body (Payload 规范):

{
    "type": "love_letter", // 对应 src/projects/ 下的文件夹名称
    "data": { // 任意传递给该模版的 JSON 数据参数
        "title": "To 某某",
        "sender": "小明",
        "receiver": "小红",
        "paragraphs": ["这是第一段话", "这是第二段话"]
    }
}

3. 前端开发：如何新增一个项目模版

如果您想为平台开发一款全新的功能页面（例如：生日祝福网页），请遵循以下目录约定：

  1. 创建目录：在 romancespace/src/projects/ 下新建您的项目文件夹。例如 birthday/。
  2. 编写主逻辑：在您的文件夹中新建 index.js 并导出一个名为 render(config) 的函数。
  3. 注册组件：前往 romancespace/src/index.js，在顶部 import 您的新目录，并将其加入到 PROJECT_REGISTRY 字典中。

src/projects/birthday/index.js (样例):

export function render(config) {
    const { name = "Friend", age = 18 } = config;
    
    // 务必返回一段完整的 HTML 字符串
    return `
        <html>
            <body>
                <h1>Happy ${age}th Birthday, ${name}!</h1>
            </body>
        </html>
    `;
}
