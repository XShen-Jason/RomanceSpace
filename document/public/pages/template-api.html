🧩 各模板定制参数 (Data) 字典

当通过 KV 接口 给用户分配或修改模板时，除了必填的 type (模板代号) 之外，最重要的是填入 data (用户定制数据)。

正如您所主张的最佳实践，将所有模板的传参说明合并在一页展示效率最高。以下列出了当前系统中所有已注册模板可接收的 data 字典。

💌 模板一：表白信 (Love Letter)

模板标识 (type): love_letter

业务场景: 用于情书展示、表白或者带信封拆解交互的单页面。

该模板可接收的 Data 属性表

字段 (Key) 类型 默认值 说明
title String "My Love Letter" 网页的标题文字
sender String "Secret Admirer" 写信人的署名
receiver String "You" 收信人的名字，通常称呼会显示在最上方
paragraphs Array<String> ["I have liked you for a long time."] 信件正文内容。数组里的每一条字符串将被渲染为一个独立段落

[推入 KV] 完整组合 JSON 示例

{
    "type": "love_letter",
    "data": {
        "title": "给最爱的红红",
        "sender": "小明",
        "receiver": "小红",
        "paragraphs": [
            "这是我遇见你的第100天。",
            "感谢命运让我们相视一笑，希望能永远陪你走下去。"
        ]
    }
}

🗓️ 模板二：恋爱纪念日 (Anniversary)

模板标识 (type): anniversary

业务场景: 用于呈现两人在一起的时光正计时、重大日期纪念大屏等。

该模板可接收的 Data 属性表

字段 (Key) 类型 默认值 说明
title String "Our Anniversary" 纪念日网页或大标题
startDate String "2023-01-01" 相识日或重要日期的起始时间(格式为 YYYY-MM-DD)，将用于计算纪念过去了多少天
names Array<String> ["Alice", "Bob"] 页面中央显示的双方姓名或昵称数组（建议长度2）
message String "Thank you for everything." 计时器底部陪伴语或宣言

[推入 KV] 完整组合 JSON 示例

{
    "type": "anniversary",
    "data": {
        "title": "小明与小红的纪念日",
        "startDate": "2021-05-20",
        "names": ["Xiaoming", "Xiaohong"],
        "message": "一直一直在一起吧、无论未来多久。"
    }
}
💡 本文档随着新模版的研发和入库会自动追加（如增加照片墙墙、烟花特效等模板后会一并更新于此）。
